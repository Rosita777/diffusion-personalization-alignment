# Prior-Compatibility Ladder for Stage 1 v2 Design

Date: 2026-06-22

Status: implemented and run as Stage 1 v2. The completed 8-subject smoke run also produced a No-Go; keep this spec as the design record, but treat the v2 result as evidence that the off-priorness metric/control construction needs revision before personalization fine-tuning.

Completed run note, 2026-06-23:

- Experiment directory: `experiments/off_prior_measurement_v0/ladder_v2/`.
- Runnable subject set: `dog`, `cat`, `backpack`, `vase`, `colorful_sneaker`, `shiny_sneaker`, `fancy_boot`, `dog7`.
- `dog7` replaced `grey_sloth_plushie` because large GitHub blob reads for the latter repeatedly stalled in this environment.
- The run used `reference_images_per_subject: 1` as a network-stable smoke setting.
- Go / No-Go result: No-Go. Hard references were positive for 3 of 8 subjects, hard references exceeded standard references for 0 of 8 subjects, standard references exceeded easy controls for 5 of 8 subjects, and the roundtrip sanity check failed.
- Next research action: revise measurement and controls, especially VAE/roundtrip confounds and natural hard-reference selection, before Stage 2.

## Purpose

The first Stage 1 smoke test showed an important negative result:

```text
ordinary DreamBooth references on the runnable lightweight subset were not
consistently more off-prior than base-generated easy controls.
```

The metric did respond to deliberately unusual hard controls, so the measurement pipeline is not useless. The failure is more likely in the experimental variable:

```text
we did not control reference prior compatibility strongly enough.
```

Stage 1 v2 therefore turns prior compatibility into an explicit ladder:

```text
easy / in-prior < standard DreamBooth < hard / off-prior
```

The goal is to test whether denoising-target residuals increase as reference images become less compatible with the pretrained SD 1.5 prior. If this ladder is visible, it becomes the evidence base for DADT target correction. If it is not visible, the project should revise the off-priorness metric before any personalization fine-tuning.

## Core Hypothesis

For matched subject class, conditioning, timestep, and noise seed:

```text
O(hard_reference) > O(standard_reference) > O(easy_control)
```

where `O` is floor-adjusted target off-priorness:

```text
O = distance(v_ref, v_base) - median_distance(base_easy_control, v_base)
```

The ladder should hold most clearly under `class` and `class_context` conditioning. Null conditioning is useful diagnostically, but it must not be the only positive result.

## Why This Is Different From Cherry-Picking

The hard reference regime is not hidden example selection. It is a declared controlled stress test:

```text
reference prior compatibility
```

The paper should say plainly:

- standard references test the classic DreamBooth setting;
- hard references test whether prior-incompatible supervision creates stronger off-prior denoising targets;
- easy controls estimate the base-model error floor;
- hard controls test whether unusual context alone is detected by the metric.

This framing makes the "small trick" scientifically defensible: we are not pretending hard examples are random; we are measuring a causal variable that personalization papers usually leave implicit.

## Experiment Groups

Each subject should produce these groups:

```text
base_easy_control
base_hard_control
dreambooth_standard_reference
dreambooth_hard_reference
vae_roundtrip_control
```

### base_easy_control

Images generated by the base SD 1.5 model from plain class prompts, such as:

```text
a photo of a dog
a photo of a backpack
a photo of a clock
```

This group estimates the residual floor of images that are, by construction, compatible with the base generator.

### base_hard_control

Images generated by SD 1.5 from unusual-context prompts, such as:

```text
a photo of a backpack half cropped in a crowded market at night
a photo of a clock in a dark forest under moonlight
```

This group is not evidence for DreamBooth forgetting by itself. It checks that the metric detects off-prior contexts when those contexts are deliberately generated.

### dreambooth_standard_reference

Original DreamBooth reference images. This group preserves comparability with DreamBooth, Preserve and Personalize, DCO, Custom Diffusion, and related personalization papers.

### dreambooth_hard_reference

A declared stress-test regime built from either naturally hard DreamBooth subjects or controlled variants of standard references.

Hardness axes:

- unusual background or scene;
- unusual lighting or global color;
- unusual crop, scale, or subject placement;
- high-frequency texture and reflections;
- strong subject-background binding;
- class-ambiguous view or pose.

The hard regime should be recorded with a `hardness_axis` field so later analysis can ask which axis drives off-priorness.

### vae_roundtrip_control

Original reference images encoded and decoded through the SD 1.5 VAE. This remains a sanity check for VAE reconstruction artifacts.

## Hard Reference Construction

Use two complementary sources.

### Natural Hard References

Some DreamBooth subjects are naturally harder because of scene, texture, crop, or subject rarity. For these, keep the original images but label the subject-level hardness axes.

This keeps the data realistic and benchmark-compatible.

### Controlled Hard Variants

For each standard reference, create deterministic variants:

```text
crop_large_subject
crop_small_subject
low_light_color_shift
high_saturation_color_shift
background_clutter_overlay
edge_reflection_texture
```

The first implementation should start with transformations that do not require segmentation:

- center crop and off-center crop;
- resized subject with padding;
- brightness, contrast, saturation, and hue shifts;
- Gaussian noise or JPEG compression stress;
- high-frequency texture overlay outside a central subject proxy region.

Segmentation-based background replacement can come later with SAM or an equivalent mask source. It is useful but not required for v2.

Each generated hard variant must be recorded with:

```text
source_standard_image
variant_id
hardness_axis
transform_parameters
```

This makes the stress test reproducible and prevents hidden cherry-picking.

Keep the existing axes:

Add required grouping fields:

```text
reference_regime:
  easy_control

  standard_reference
  hard_reference
  roundtrip_control

hardness_axis:
  none
  natural_hard_subject
  crop
  color_light
  high_frequency
  clutter_background
  subject_background_binding
```

The manifest should preserve `null` as a string, not as CSV NaN. The current `csv_io.py` fix must be reused.

## Metrics

Keep the current global residuals:

```text
normalized_l2
cosine_distance
base_to_ref_norm_ratio
floor_adjusted_l2
```

Report these summaries:

- per subject;
- per reference regime;
- per hardness axis;
- per timestep;
- per frequency band;
- per conditioning key.

Add ladder-specific scores:

```text
ladder_gap_standard = mean(O(standard_reference)) - mean(O(easy_control))
ladder_gap_hard = mean(O(hard_reference)) - mean(O(standard_reference))
ladder_monotonic = O(hard_reference) > O(standard_reference) > O(easy_control)
```

The main v2 figure should be:

```text
reference_regime x timestep heatmap of mean floor_adjusted_l2
```

The second figure should be:

```text
hardness_axis x frequency_band heatmap
```

## Go / No-Go Rule

Proceed to Stage 2 only if all of the following hold:

- `hard_reference` has positive mean floor-adjusted residual under `class` or `class_context`.
- At least 6 of 8 v2 subjects have `hard_reference > standard_reference`.
- At least 4 of 8 v2 subjects have `standard_reference > easy_control`, or the paper explicitly narrows the claim to hard-reference personalization.
- `base_hard_control` remains positive, confirming the metric detects off-prior context.
- `vae_roundtrip_control` stays close to `standard_reference` or below it, showing the result is not only a VAE artifact.

If only hard references are positive, the claim should become:

```text
off-prior personalization references can induce target distribution gap
```

not:

```text
all ordinary DreamBooth references are off-prior
```

## Expected Reviewer-Facing Story

The first smoke test becomes a useful negative result:

```text
ordinary references are not automatically off-prior.
```

The v2 ladder should then establish the sharper claim:

```text
reference prior compatibility controls target off-priorness.
```

This is stronger than the original broad hypothesis because it explains when target alignment is needed. It also naturally motivates a selective method:

- if `O` is low, trust the reference target;
- if `O` is high in background or low-frequency bands, align toward the base field;
- if `O` is high only in subject high-frequency bands, preserve subject identity.

## Risks

- If hard references are generated by crude transformations, reviewers may call them artificial. Mitigation: include both natural hard subjects and controlled variants.
- If only base hard controls are positive, the metric detects odd prompts but not reference supervision. Mitigation: revise region or frequency metrics before Stage 2.
- If hard references improve off-priorness but harm subject identity too much, the method must separate prior-harmful background deviations from identity-carrying details.
- If dataset access remains unstable, document local dataset placement and manifest checks before running more experiments.
